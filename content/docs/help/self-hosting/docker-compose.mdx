Most Omni products follow a standard Docker Compose pattern for self-hosting. This page covers the general approach and product-specific quickstarts.

## Standard Pattern

Every Docker-compatible Omni product ships a `compose.yaml` in its stack repository. Most also include a `.env.local.template` with required and optional environment variables:

```bash
# Clone the stack repo
git clone https://github.com/omnidotdev/<product>-stack.git
cd <product>-stack

# Copy env template (if present) and configure
cp .env.local.template .env.local
# Edit .env.local with your settings

# Run
docker compose --env-file .env.local up -d
```

If a stack doesn't include `.env.local.template`, check the `compose.yaml` for required environment variables and create `.env.local` manually.

## Building from Source

To build images locally instead of pulling pre-built images, use the development override:

```bash
git clone https://github.com/omnidotdev/<product>-stack.git
cd <product>-stack

cp .env.local.template .env.local
docker compose --env-file .env.local -f compose.yaml -f compose.dev.yaml up --build
```

## Product Quickstarts

### Runa

[Runa](/core/runa) runs three containers:

| Container | Image | Port |
|-----------|-------|------|
| `db` | `postgres:16-alpine` | 5432 |
| `api` | `manifold.omni.dev/omnidotdev/runa-api:latest` | 4000 |
| `app` | `manifold.omni.dev/omnidotdev/runa-app:latest` | 3000 |

```bash
git clone https://github.com/omnidotdev/runa-stack.git
cd runa-stack

cp .env.local.template .env.local
# Set DB_PASSWORD, AUTH_SECRET, AUTH_CLIENT_ID, AUTH_CLIENT_SECRET

docker compose --env-file .env.local up -d
```

### Vortex

[Vortex](/grid/vortex) runs six containers:

| Container | Image | Port |
|-----------|-------|------|
| `db` | `postgres:16-alpine` | 5432 |
| `cache` | `valkey/valkey:8-alpine` | 6379 |
| `events` | `apache/iggy:0.7.0` | 8090 (TCP) |
| `api` | `manifold.omni.dev/omnidotdev/vortex-api:latest` | 4000 |
| `app` | `manifold.omni.dev/omnidotdev/vortex-app:latest` | 3000 |
| `worker` | `manifold.omni.dev/omnidotdev/vortex-worker:latest` | 8080 (internal) |

Health checks are configured for all services. The API waits for PostgreSQL, Valkey, and Iggy to be healthy before starting.

<Callout type="info">
  Iggy requires elevated capabilities for optimal performance: `SYS_NICE` (cap_add), `seccomp:unconfined`, and unlimited `memlock` ulimits. These are pre-configured in the default `compose.yaml`. If your container runtime restricts these, Iggy may still start but with degraded throughput.
</Callout>

```bash
git clone https://github.com/omnidotdev/vortex-stack.git
cd vortex-stack

cp .env.local.template .env.local
# Set DB_PASSWORD, AUTH_SECRET, CACHE_PASSWORD, IGGY_USERNAME, IGGY_PASSWORD

docker compose up -d
```

#### Required environment variables

| Variable | Description |
|----------|-------------|
| `DB_PASSWORD` | PostgreSQL password |
| `AUTH_SECRET` | Better Auth session secret |
| `CACHE_PASSWORD` | Valkey password |
| `IGGY_USERNAME` | Iggy admin username |
| `IGGY_PASSWORD` | Iggy admin password |

Optional: `DB_NAME`, `DB_USER`, `DB_PORT`, `API_PORT`, `APP_PORT`, `CACHE_PORT`, and OAuth provider credentials (`GOOGLE_*`, `GITHUB_*`, `OIDC_*`). See the full [Vortex configuration reference](/grid/vortex/configuration).

#### Volumes

Three named volumes persist data across restarts:

| Volume | Backing service | Content |
|--------|----------------|---------|
| `db_data` | PostgreSQL | Database files |
| `cache_data` | Valkey | Cache state |
| `events_data` | Iggy | Event stream segments |

### Manifold

[Manifold](/armory/manifold) runs three containers:

| Container | Port | Description |
|-----------|------|-------------|
| `db` | 5432 | PostgreSQL |
| `api` | 4000 | Manifold API |
| `app` | 3000 | Manifold web UI |

```bash
git clone https://github.com/omnidotdev/manifold-stack.git
cd manifold-stack

# Configure required env vars (DB_PASSWORD)
docker compose --env-file .env.local up -d
```

#### Required environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DB_PASSWORD` | Yes | PostgreSQL password |
| `CORS_ALLOWED_ORIGINS` | No | Comma-separated allowed origins |
| `AUTH_BASE_URL` | No | OIDC provider base URL |
| `VITE_API_BASE_URL` | No | API URL for the web app (default: `http://api:4000`) |

#### Storage backend

Manifold supports filesystem or S3-compatible blob storage:

| Variable | Default | Description |
|----------|---------|-------------|
| `BLOB_STORAGE_BACKEND` | `fs` | Storage backend: `fs` or `s3` |
| `BLOB_STORAGE_DIR` | `/data/manifold/blobs` | Filesystem storage base directory |
| `BLOB_S3_BUCKET` | — | S3 bucket name (required when backend is `s3`) |
| `BLOB_S3_REGION` | `us-east-1` | S3 region |
| `BLOB_S3_ENDPOINT` | — | Custom S3 endpoint (for MinIO, R2) |
| `BLOB_S3_PREFIX` | `blobs` | S3 key prefix |

See the [Manifold Storage guide](/armory/manifold/storage) for details.

### Synapse

[Synapse](/grid/synapse) runs five containers:

| Container | Image | Port |
|-----------|-------|------|
| `db` | `postgres:16-alpine` | 5432 |
| `valkey` | `valkey/valkey:8-alpine` | 6379 |
| `gateway` | `manifold.omni.dev/omnidotdev/synapse-gateway:latest` | 6000 |
| `api` | `manifold.omni.dev/omnidotdev/synapse-api:latest` | 4000 |
| `app` | `manifold.omni.dev/omnidotdev/synapse-app:latest` | 3000 |

```bash
git clone https://github.com/omnidotdev/synapse-stack.git
cd synapse-stack

cp .env.example .env.local
# Set DB_PASSWORD, AUTH_SECRET, GATEWAY_SECRET, ENCRYPTION_KEY, and at least one LLM provider key

docker compose --env-file .env.local up -d
```

#### Required environment variables

| Variable | Description |
|----------|-------------|
| `DB_PASSWORD` | PostgreSQL password |
| `AUTH_SECRET` | Session signing secret |
| `GATEWAY_SECRET` | Internal gateway auth secret |
| `ENCRYPTION_KEY` | Provider key encryption at rest |

Optional: `AUTH_BASE_URL` (HIDRA Gatekeeper), `BILLING_*` (billing), `CORS_ALLOWED_ORIGINS`, port overrides. Set `PROTECT_ROUTES=false` to run without an identity provider.

#### Volumes

| Volume | Backing service | Content |
|--------|----------------|---------|
| `db_data` | PostgreSQL | Database files |
| `valkey_data` | Valkey | Rate limit state and cache |

See the [Synapse self-hosting guide](/grid/synapse/self-hosting) for Helm charts, gateway configuration, and binary installs.

## Database Migrations

Products that use PostgreSQL run database migrations automatically on API startup. No manual migration step is required.
