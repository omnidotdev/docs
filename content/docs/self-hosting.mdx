Many Omni products support self-hosting with Docker.

## Quick Start

Using [ðŸŒ™ Runa](/core/runa) as an example:

```sh
# Download Compose files
curl -sLO https://raw.githubusercontent.com/omnidotdev/runa/master/compose.yaml
curl -sLO https://raw.githubusercontent.com/omnidotdev/runa/master/.env.local.template

# Configure
cp .env.local.template .env.local
# Edit .env.local with your settings

# Run
docker compose --env-file .env.local up
# Or run in background: docker compose --env-file .env.local up -d
```

## Development Mode

To build from source instead of pulling pre-built images:

```sh
git clone https://github.com/omnidotdev/runa.git
cd runa

cp .env.local.template .env.local
docker compose --env-file .env.local -f compose.yaml -f compose.dev.yaml up
```

## Upgrading

```sh
docker compose pull
docker compose --env-file .env.local up -d
```

## Authentication

Email/password authentication is enabled by default. OAuth providers activate automatically when credentials are provided:

| Provider | Environment Variables |
|----------|----------------------|
| Google | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` |
| GitHub | `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` |
| Generic OIDC | `OIDC_ISSUER`, `OIDC_CLIENT_ID`, `OIDC_CLIENT_SECRET` |

`AUTH_SECRET` is always required for session encryption. Generate one with:

```sh
openssl rand -base64 32
```

### Google OAuth Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com/apis/credentials)
2. Create OAuth 2.0 Client ID (Web application)
3. Add authorized redirect URI: `https://your-domain.com/api/auth/callback/google`
4. Copy Client ID and Secret to your `.env.local`

### GitHub OAuth Setup

1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
2. Create new OAuth App
3. Set callback URL: `https://your-domain.com/api/auth/callback/github`
4. Copy Client ID and Secret to your `.env.local`

### Generic OIDC (Enterprise SSO)

For enterprise identity providers (Okta, Azure AD, Keycloak, etc.):

```sh
OIDC_ISSUER=https://your-idp.com
OIDC_CLIENT_ID=your-client-id
OIDC_CLIENT_SECRET=your-client-secret
```

The issuer URL should support OIDC discovery (`/.well-known/openid-configuration`).

## Backup

```sh
# Database backup
docker compose exec db pg_dump -U postgres runa > backup.sql

# Restore
docker compose exec -T db psql -U postgres runa < backup.sql
```
