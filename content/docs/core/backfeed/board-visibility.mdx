---
title: Board Visibility
description: Public and private boards, and what each visibility mode means
---

Backfeed projects are **public by default** — this is intentional, since the primary use case is collecting feedback from external users who may not have accounts.

Admins can toggle any project to **private** in **Project Settings → Visibility**. When a project is private:

- Only authenticated organization members can access it
- Unauthenticated visitors see a 404
- Useful for internal feedback boards or pre-launch projects

When a project is public:

- Anyone can view the board (and the roadmap and changelog, when enabled) without an account
- Submitting a post or voting requires signing in
- The project appears at its shareable workspace URL
