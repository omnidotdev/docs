**Vortex** is an open source workflow automation platform. Build, deploy, and monitor workflows with a visual drag-and-drop editor or code. Connect 500+ services, schedule tasks, react to events, and orchestrate complex multi-step processes with built-in retries and error handling.

## Key Features

- **Visual Workflow Builder**: Design workflows with an intuitive drag-and-drop editor. No coding required to get started.
- **500+ Integrations**: Connect services like Slack, GitHub, Stripe, Discord, Google Sheets, and hundreds more.
- **Durable Execution**: Built-in retries, error handling, dead letter queues, and monitoring. Workflows run reliably.
- **Multiple Trigger Types**: Manual, cron schedule, webhook, polling, CloudEvents, and SSE triggers.
- **CloudEvents Native**: First-class support for the CloudEvents specification. Publish and subscribe to events across services.
- **Custom Plugins**: Extend Vortex with WebAssembly plugins for custom workflow steps (Pro plan).
- **Self-Hosting**: Run Vortex on your own infrastructure with Docker Compose or Kubernetes.

## Architecture

Vortex consists of three services:

| Service | Purpose | Stack |
|---------|---------|-------|
| **vortex-api** | GraphQL + REST API, authentication, entitlements | Bun, Elysia, PostGraphile, Drizzle, PostgreSQL |
| **vortex-app** | Visual workflow editor, dashboard, monitoring | React 19, TanStack Router, ReactFlow, Tailwind |
| **vortex-worker** | DSL executor, integration connectors, trigger adapters | Bun, Hatchet, ActivePieces connectors |

## Getting Started

1. **Sign up** at [vortex.omni.dev](https://vortex.omni.dev) (free, no credit card required)
2. **Create a workspace** from the dashboard
3. **Build a workflow** using the visual editor or start from a template
4. **Connect integrations** (Slack, GitHub, etc.) via OAuth
5. **Execute and monitor** your workflow from the dashboard

## Self-Hosting

Clone the repository and run with Docker Compose:

```bash
git clone https://github.com/omnidotdev/vortex.git
cd vortex
cp .env.local.template .env.local
# Edit .env.local with your configuration
docker compose up -d
```

See the [repository README](https://github.com/omnidotdev/vortex) for full self-hosting documentation.
