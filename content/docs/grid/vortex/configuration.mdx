Vortex is composed of three application services (API, app, worker) and three infrastructure dependencies (PostgreSQL, Valkey, Iggy). Each service is configured via environment variables.

## Infrastructure

| Service | Default port | Purpose |
|---|---|---|
| **PostgreSQL 18** | 5432 | Primary data store (workflows, runs, credentials, routing rules) |
| **Valkey 8** | 6379 | Distributed cron locking and caching (Redis-compatible) |
| **Apache Iggy** | 8090 (TCP) | Event streaming (OmniEvent bus) |

## API (`vortex-api`)

The API handles GraphQL/REST requests, cron scheduling, event routing, and workflow dispatch.

### Required

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `AUTH_BASE_URL` | OIDC identity provider URL |
| `CORS_ALLOWED_ORIGINS` | Comma-separated allowed origins |
| `ENCRYPTION_KEY` | Base64-encoded 32-byte key for encrypting stored integration credentials |
| `HATCHET_CLIENT_TOKEN` | Token for the Hatchet workflow engine (required when using the default Hatchet executor) |
| `INTERNAL_API_SECRET` | Shared secret for internal service-to-service calls |
| `IGGY_USERNAME` | Iggy authentication username |
| `IGGY_PASSWORD` | Iggy authentication password |

### Optional

| Variable | Default | Description |
|---|---|---|
| `CACHE_URL` | - | Valkey/Redis URL for distributed locking (required for multi-instance) |
| `EVENTS_URL` | - | Iggy TCP connection string |
| `WORKER_URL` | - | Worker service URL for health forwarding |
| `VORTEX_PUBLIC_URL` | - | Public-facing Vortex URL (used for OAuth callbacks) |
| `PLATFORM_ORG_ID` | - | Platform organization ID (multi-tenant deployments) |
| `BILLING_BASE_URL` | - | Aether billing service URL |
| `BILLING_WEBHOOK_SECRET` | - | Aether billing webhook HMAC secret |
| `BILLING_SERVICE_API_KEY` | - | Aether billing service API key |
| `IDP_WEBHOOK_SECRET` | - | Gatekeeper IDP webhook HMAC secret |
| `AUTHZ_API_URL` | - | Warden AuthZ API URL (enables AuthZ when set) |
| `AUTHZ_WEBHOOK_SECRET` | - | Warden AuthZ webhook HMAC secret |
| `AUTH_WEBHOOK_SECRET` | - | Auth service webhook HMAC secret |
| `EMAIL_WEBHOOK_SECRET` | - | Email service webhook HMAC secret |
| `STRIPE_API_KEY` | - | Stripe secret key |
| `STRIPE_WEBHOOK_SECRET` | - | Stripe webhook signing secret |
| `TEMPORAL_ADDRESS` | - | Temporal server address (when using Temporal backend) |
| `TEMPORAL_NAMESPACE` | - | Temporal namespace |
| `TEMPORAL_TASK_QUEUE` | - | Temporal task queue |
| `GRAPHQL_MAX_COMPLEXITY_COST` | `5000` | Maximum GraphQL query cost |
| `PROTECT_ROUTES` | `true` | Enable auth middleware on API routes |
| `AUTH_DEBUG` | - | Enable Better Auth debug logging |
| `LOG_LEVEL` | - | Log level override |

|---|---|---|

## App (`vortex-app`)

The app is the React dashboard with visual workflow editor, user management, and real-time subscriptions.

### Required

| Variable | Description |
|---|---|
| `AUTH_SECRET` | Session signing secret (generate with `openssl rand -base64 32`) |
| `VITE_BASE_URL` | Public app URL (e.g., `https://vortex.omni.dev`) |
| `VITE_API_BASE_URL` | Vortex API base URL |
| `VITE_AUTH_BASE_URL` | OIDC identity provider URL |

### Optional

| Variable | Default | Description |
|---|---|---|
| `AUTH_CLIENT_ID` | - | OIDC client ID |
| `AUTH_CLIENT_SECRET` | - | OIDC client secret |
| `CACHE_URL` | - | Valkey/Redis URL (persists sessions across pod restarts) |
| `API_INTERNAL_URL` | `VITE_API_BASE_URL` | Internal API URL for server-to-server communication (e.g., Docker service name) |
| `VITE_BILLING_BASE_URL` | - | Aether billing API URL |
| `VITE_AUTHZ_API_URL` | - | Warden AuthZ API URL |
| `VITE_FLAGS_CLIENT_KEY` | - | Unleash feature flag client key |
| `VITE_FLAGS_API_HOST` | - | Unleash API host URL |

### OAuth providers (user login)

All OAuth providers are optional. Configure any combination for login:

| Variable | Description |
|---|---|
| `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | Google OAuth |
| `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` | GitHub OAuth |
| `OIDC_ISSUER`, `OIDC_CLIENT_ID`, `OIDC_CLIENT_SECRET` | Generic OIDC (e.g., Keycloak, Auth0, HIDRA) |

## Worker (`vortex-worker`)

The worker executes workflow DSL definitions, manages trigger adapters, and runs integration connectors.

### Required

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `HATCHET_CLIENT_TOKEN` | Hatchet workflow engine token (required when using the default Hatchet executor) |
| `INTERNAL_API_SECRET` | Shared secret for internal service-to-service calls |

### Optional

| Variable | Default | Description |
|---|---|---|
| `VORTEX_EXECUTOR` | `hatchet` | Workflow executor backend (`hatchet` or `temporal`) |
| `CACHE_URL` | - | Valkey/Redis URL |
| `EVENTS_URL` | - | Iggy TCP connection string |
| `IGGY_USERNAME` | - | Iggy authentication username |
| `IGGY_PASSWORD` | - | Iggy authentication password |
| `HATCHET_CLIENT_TLS_STRATEGY` | `none` | TLS mode for Hatchet connection |
| `ENCRYPTION_KEY` | - | Base64-encoded 32-byte key for credential decryption |
| `TEMPORAL_ADDRESS` | - | Temporal server address (when using Temporal backend) |
| `TEMPORAL_NAMESPACE` | - | Temporal namespace |
| `TEMPORAL_TASK_QUEUE` | - | Temporal task queue |
| `VORTEX_API_URL` | - | API service URL (for internal callbacks) |
| `VORTEX_PUBLIC_URL` | - | Public-facing Vortex URL |
| `VORTEX_CALLBACK_BASE_URL` | - | Base URL for OAuth callback redirects |
| `VORTEX_TRIGGER_PLUGINS_DIR` | - | Directory for external trigger adapter plugins |
| `HEALTH_PORT` | `8080` | Port for the health check HTTP server |
| `LOG_LEVEL` | - | Log level override |

## Generating secrets

```bash
# AUTH_SECRET (app session signing)
openssl rand -hex 16

# ENCRYPTION_KEY (base64-encoded 32 bytes)
openssl rand -base64 32

# INTERNAL_API_SECRET (service-to-service auth)
openssl rand -hex 16

# DB_PASSWORD / CACHE_PASSWORD (for Docker Compose)
openssl rand -hex 16
```

## Health checks

| Service | Endpoint | Port | Purpose |
|---|---|---|---|
| API | `/health` | 4000 | Liveness, returns `{ status, timestamp, service }` |
| API | `/ready` | 4000 | Readiness, verifies DB and cache connectivity |
| App | `/api/_health` | 3000 | Liveness |
| Worker | `/health` | 8080 | Liveness |

## Rate limiting

The API enforces a default rate limit of **100 requests per minute per IP**. GraphQL queries are cost-limited to a configurable maximum (`GRAPHQL_MAX_COMPLEXITY_COST`, default 5000).
