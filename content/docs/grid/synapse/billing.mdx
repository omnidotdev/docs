Synapse offers flexible billing through [Aether](https://billing.omni.dev), Omni's billing service. You can bring your own provider keys (BYOK), use managed credits, or combine both.

## Billing Modes

### BYOK (Bring Your Own Key)

Store your own provider API keys in Synapse's encrypted vault. Requests route directly through your provider accounts. You pay providers directly and Synapse charges only for the platform subscription.

### Managed Credits

Use Synapse-provided keys with pay-as-you-go pricing. Provider costs are passed through with a 15% margin. No need to manage individual provider accounts.

### Hybrid

Combine BYOK and managed keys. Attach provider keys to specific Synapse API keys, and fall back to managed credits when no BYOK key is available.

## Entitlements

Each tier includes a set of entitlements enforced by the gateway and API:

- **`api_access`** -- whether API requests are allowed
- **`max_requests_per_month`** -- request quota (-1 = unlimited)
- **`max_api_keys`** -- API key creation limit (-1 = unlimited)
- **`max_input_tokens_per_request`** -- per-request input token cap
- **`max_output_tokens_per_request`** -- per-request output token cap
- **`smart_routing`** -- access to cost/quality/latency routing strategies
- **`stt_enabled`**, **`tts_enabled`**, **`image_gen_enabled`**, **`embeddings_enabled`** -- modality access
- **`analytics_retention_days`** -- how long usage data is retained
- **`team_rbac`** -- workspace roles and permissions
- **`byok_enabled`** -- bring your own provider keys

View your current entitlements in the [Synapse dashboard](https://synapse.omni.dev/dashboard/billing).

## Usage Metering

Every request through the gateway is metered:

1. **Input tokens** -- tokens sent to the provider
2. **Output tokens** -- tokens received from the provider
3. **Request count** -- incremented per API call
4. **Message count** -- number of messages in conversation

Usage is recorded asynchronously and never blocks request handling. Meter data flows to Aether for quota tracking and billing.

## Webhooks

Synapse receives entitlement updates from Aether via HMAC-signed webhooks:

- `entitlement.created` -- new subscription activated
- `entitlement.updated` -- plan changed or renewed
- `entitlement.deleted` -- subscription canceled

Plan changes take effect immediately. The billing cache is invalidated on each webhook event.

## Self-Hosted Billing

When self-hosting, billing integration is optional. Without Aether configured:

- All users default to a generous fallback quota
- BYOK mode works without any billing setup
- Usage events are still recorded locally in the database
- Set `BILLING_BASE_URL` to connect to your own Aether instance

See the [self-hosting guide](/grid/synapse/self-hosting) for configuration details.
