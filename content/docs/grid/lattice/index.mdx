**Omni Lattice** is a GraphQL Federation gateway that composes multiple subgraph services into a unified supergraph API. Built from the ground up in Rust with an MIT license.

## Quick Start

### 1. Install

```bash
# Build from source
cargo install --git https://github.com/omnidotdev/lattice lattice-cli
cargo install --git https://github.com/omnidotdev/lattice lattice

# Or use Docker
docker pull manifold.omni.dev/omnidotdev/lattice:latest
```

### 2. Create Config

Lattice supports **YAML**, **JSON**, and **JSONC** (JSON with comments) config formats. Create `lattice.yaml`:

```yaml
server:
  listen: "0.0.0.0:4000"
  playground: true

supergraph:
  path: ./supergraph.graphql

subgraphs:
  aether:
    url: http://localhost:4001/graphql
  warden:
    url: http://localhost:4002/graphql
```

### 3. Compose Supergraph

```bash
lattice compose --config lattice.yaml --output supergraph.graphql
```

This introspects each subgraph and composes them into a federated supergraph schema.

### 4. Run Gateway

```bash
lattice serve --config lattice.yaml
```

Your unified GraphQL API is now available at `http://localhost:4000` with the GraphiQL playground at the root.

## Docker Deployment

```bash
docker run -p 4000:4000 \
  -v ./lattice.yaml:/etc/lattice/lattice.yaml \
  -v ./supergraph.graphql:/etc/lattice/supergraph.graphql \
  manifold.omni.dev/omnidotdev/lattice
```

## Key Features

- **GraphQL Federation v2**: Query planning and execution across subgraphs
- **`@defer` Support**: Incremental delivery via `multipart/mixed` responses
- **Request Deduplication**: Coalesce identical in-flight subgraph fetches
- **Directive-Based Authorization**: Enforce `@requiresScopes` and `@requiresAuthentication`
- **WebSocket Subscriptions**: Real-time data via `graphql-transport-ws` protocol
- **JWT Authentication**: JWKS-based token validation with multi-provider support
- **Entity Caching**: Valkey/Redis-backed response caching
- **Rate Limiting**: Per-client request throttling
- **WASI Plugins**: 9 hook points via Extism (request, parse, validation, plan, subgraph, response)
- **OpenTelemetry**: Distributed tracing and observability
- **GraphiQL Playground**: Built-in IDE at `GET /`
- **Native Schema Composition**: Built-in `lattice compose` command
- **Multiple Config Formats**: YAML, JSON, and JSONC support
- **In-Process Subgraphs**: Run subgraphs without HTTP via `SubgraphExecutor` trait
- **Environment Variables**: `${VAR}` substitution in config files
- **High Performance**: Written in Rust for minimal latency
