**Omni Beacon** is a local-first AI assistant gateway built in Rust. It ships as a single binary with an embedded AI router ([Synapse](/grid/synapse)) — no separate server needed. It runs on your machine or home server, keeping all your data local while providing seamless multi-device access through device-based identity and mDNS discovery.

Beacon goes beyond simple chat — it's a complete platform for running AI assistants across voice, 14+ messaging channels, and a WebSocket API, with agentic tool execution, semantic memory, rule-based routing, attachment processing, and extensible skills.

## Local-First Principles

Beacon is designed around local-first principles:

- **Gateway is the source of truth**: All data lives on your gateway
- **No cloud account required**: Works completely offline
- **Device identity, not user accounts**: Ed25519 keypairs per device
- **mDNS auto-discovery**: Apps find your gateway automatically on LAN
- **Optional cloud sync**: Connect to Omni for cross-device memory sync

## Chat

Talk to your AI assistant through a familiar chat interface or the WebSocket API. Beacon keeps your conversation history locally in SQLite, with support for sessions, threading, and long-term semantic memory with vector search. Synapse runs in-process by default, routing requests to your configured provider. Or use [Omni Credits](/grid/beacon/cloud-features) for a zero-config experience.

## Agents

Beacon supports a multi-agent architecture with pluggable execution backends:

- **Built-in agent**: Runs agentic tool-use loops directly via Synapse with configurable model, system prompt, max tokens, and iteration limits
- **Harness adapters**: Delegate agent execution to external CLI tools (e.g. Claude Code) with session resumption, MCP server config, and environment variable passthrough
- **Agent registry**: Register multiple named agents and route messages to specific agents via routing rules
- **Agent binding**: Route messages to the appropriate agent based on context

## Voice

Beacon supports hands-free interaction with wake word detection, speech-to-text, and text-to-speech. Say your persona's wake word, ask a question, and hear the answer spoken back. See the [Voice guide](/grid/beacon/voice) for setup details.

| Component | Local | Embedded (via [Synapse](/grid/synapse)) |
|-----------|-------|----------------------------------------|
| Wake word detection | Yes | - |
| Speech-to-text | - | Whisper, Deepgram |
| Text-to-speech | - | OpenAI TTS, ElevenLabs |

## Personas

Beacon uses configurable [personas](/grid/beacon/personas) to shape how your assistant sounds and behaves. The default persona is [Orin](/orin), a friendly otter guide, but you can create your own or install community personas from [Manifold](https://manifold.omni.dev). Each persona has its own wake word, voice, and personality. See the [persona.json spec](/codex/persona-json) for advanced customization.

## Channels

Connect Beacon to 14+ messaging platforms. Your assistant works consistently across Discord, Slack, Telegram, Teams, Signal, Gmail, IRC, Twilio (SMS), WhatsApp, Matrix, Google Chat, iMessage, Feishu/Lark, and Line. Each channel declares its own capabilities (reactions, streaming, media, etc.). See [Channels](/grid/beacon/channels) for the full list and setup instructions.

## Routing

Rule-based message routing lets you control how incoming messages are handled before they reach the agent:

- **Allow/deny rules** with priority ordering
- **Match conditions**: filter by channel, sender ID, content pattern (regex), or DM-only
- **Actions**: allow, deny, deny with reply, redirect to agent, or rate limit
- **Per-sender rate limiting** with rolling 60-second windows

## Attachments

Beacon processes attachments inline during the message flow:

- **Vision**: Image and video analysis via configurable vision providers
- **PDF**: Text extraction from PDF attachments
- **Video**: Keyframe extraction (configurable max frames)
- **Media limits**: Configurable max image size (default 10 MB) and video frame count

## Memory

Semantic memory powered by vector embeddings (via OpenAI or compatible providers):

- **Post-turn extraction**: Automatically extracts and stores relevant memories after each conversation turn
- **Vector search**: Retrieve contextually relevant memories using `sqlite-vec`
- **Memory sync**: Optionally sync memories across devices via Omni cloud

## Getting Started

1. **Build**: Build the gateway from source with `cargo build --release`
2. **Set up AI**: Run `beacon setup` or configure `~/.config/omni/beacon/config.toml`
3. **Run**: Start the gateway with `beacon --foreground -v`
4. **Customize**: Pick a persona, set your wake word, and configure voice
5. **Start talking**: Say your wake word and go

See the [Getting Started guide](/grid/beacon/getting-started) for a detailed walkthrough.
