Get up and running with Beacon in a few minutes.

## Quick Start

Build and run the gateway from source:

```bash
cd services/beacon
cargo build --release
cargo run -- setup
cargo run -- --foreground -v
```

`beacon setup` walks you through provider, API key, model, and voice configuration interactively, writing `~/.config/omni/beacon/config.toml`. After setup, run `beacon --foreground -v` to start the gateway.

<Callout type="info">
  A desktop app and prebuilt binaries are planned but not yet available. For now, build from source.
</Callout>

## First Launch

When you run Beacon for the first time:

1. Your device generates a unique cryptographic identity (no account needed)
2. The gateway starts listening on `http://localhost:18789`
3. Connect via the WebSocket API or a client app

## Set Up Your AI Provider

Beacon includes [Synapse](/grid/synapse) as an embedded AI router — no separate service to install or manage. Configure your provider and API key via `beacon setup` or by editing `~/.config/omni/beacon/config.toml` directly.

### Bring Your Own Key (BYOK)

Synapse dispatches requests to the appropriate provider based on your config:

| Provider | What It Powers |
|----------|---------------|
| Anthropic | Chat (Claude) |
| OpenAI | Chat (GPT), voice transcription (Whisper), speech (TTS) |
| Deepgram | Voice transcription (STT) |
| ElevenLabs | Speech (TTS) |

### Omni Credits

Don't want to manage API keys? Connect your Omni account and use pay-per-use credits instead. Go to **Settings → Cloud → Connect Omni Account**. See [Cloud Features](/grid/beacon/cloud-features) for details.

## Pair Additional Devices

Add your phone, tablet, or another computer to your gateway:

1. On your gateway device, go to **Settings → Devices → Generate Pairing Code**
2. A 6-digit code appears (valid for 10 minutes)
3. On the new device, open Beacon and enter the code
4. The device is now paired and can chat with your assistant

You can manage paired devices and revoke access anytime from **Settings → Devices**. See [Device Identity](/grid/beacon/device-identity) for technical details.

## Try Your First Chat

Type a message in the chat box and press enter. Your assistant responds using whichever AI provider you configured.

## Try Voice

1. Make sure Synapse has an STT and TTS provider configured (e.g. OpenAI for both Whisper and TTS)
2. Say your persona's wake word — the default is **"Hey Orin"**
3. Ask a question
4. Beacon transcribes your speech, sends it to the AI, and speaks the response back

See the [Voice guide](/grid/beacon/voice) for more on wake words, provider options, and troubleshooting.

## Standalone Gateway

Run the gateway on a server or Raspberry Pi:

```bash
# Build from source
cd services/beacon
cargo build --release

# Interactive setup (provider, API key, model, voice)
./target/release/beacon setup

# Run with verbose logging
./target/release/beacon --foreground -v
```

Clients can discover the gateway automatically via [mDNS](/grid/beacon/discovery), or connect by entering the gateway's IP address manually.
