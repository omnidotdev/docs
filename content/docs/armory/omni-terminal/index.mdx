## Overview

**Your horizon to the universe.**

Omni Terminal is a GPU-accelerated terminal emulator built with Rust and WebGPU. It runs on desktop (macOS, Linux, Windows), web (WASM + WebSocket PTY server), and mobile (Android) — all from a single codebase.

## Key Features

- **GPU-accelerated rendering**: Powered by WebGPU via the Sugarloaf rendering engine
- **Tabs**: Multiple tab modes (top, bottom, bookmark, native macOS) with color automation
- **Split panes**: Horizontal and vertical splits with resizable dividers
- **VI mode**: Full VI keybindings for navigation and selection
- **Search**: Forward/backward regex search with history
- **Hints**: URL detection and opening with customizable patterns
- **Runtime opacity**: Adjust window transparency on the fly with keybindings
- **Background images**: Support for background images with opacity blending
- **Customizable keybindings**: Override any default binding through config
- **Cross-platform**: Native rendering on macOS, Linux, Windows, web, and Android
- **Web terminal**: Self-hostable web terminal with WebSocket PTY bridge and TLS
- **Android terminal**: Full Arch Linux environment via PRoot with local filesystem access

| Debian/Ubuntu | .deb | Download x11 or wayland .deb from [releases](https://github.com/omnidotdev/terminal/releases) |

### Build from Source

```bash
# Linux: install system dependencies first
# Debian/Ubuntu: sudo apt install libwayland-dev libxkbcommon-dev libfontconfig-dev
# Arch: sudo pacman -S fontconfig freetype2 libxkbcommon wayland

git clone https://github.com/omnidotdev/terminal
cd terminal
cargo build --release -p omni-terminal
```

## Quick Start

Launch the terminal:

```bash
omni-terminal
```

Start in a specific directory:

```bash
omni-terminal -w ~/projects
```

Run a command directly:

```bash
omni-terminal -e htop
```

## Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                        Frontends                             │
│  ┌──────────────┐  ┌───────────┐  ┌────────────────────┐    │
│  │ omni-terminal│  │   WASM    │  │  android-lib (JNI) │    │
│  │  (desktop)   │  │(browser)  │  │    + Kotlin app    │    │
│  └──────┬───────┘  └─────┬─────┘  └─────────┬──────────┘    │
├─────────┴────────────────┴───────────────────┴───────────────┤
│                    terminal-backend                          │
│            (PTY, grid, ANSI parsing, events)                 │
├──────────────────────────────────────────────────────────────┤
│                      Sugarloaf                               │
│          (WebGPU rendering, font rasterization)              │
├──────────────────────────────────────────────────────────────┤
│  terminal-window │ copa (parser) │ teletypewriter │corcovado │
└──────────────────────────────────────────────────────────────┘
```

| Crate | Purpose |
|-------|---------|
| **omni-terminal** | Desktop frontend — windowing, input, keybindings, tabs, splits |
| **omni-terminal-wasm** | Web frontend — WebGPU rendering in browser, WebSocket PTY client |
| **omni-terminal-android** | Android frontend — JNI bridge, PRoot PTY spawning, TLS/WSS client |
| **web-server** | Web PTY server — Axum WebSocket bridge with TLS, session management |
| **terminal-backend** | Terminal emulation — PTY management, ANSI parsing, grid state, events |
| **sugarloaf** | GPU rendering engine — WebGPU text and shape rendering, font loading |
| **terminal-window** | Window abstraction — cross-platform windowing (winit fork) |
| **copa** | Terminal parser — high-performance ANSI/VT100 sequence parsing |
| **teletypewriter** | PTY creation — pseudoterminal allocation and shell spawning |
| **corcovado** | Non-blocking I/O — cross-platform async primitives |

## CLI Arguments

| Argument | Description |
|----------|-------------|
| `-e, --command` | Command and args to execute (must be last) |
| `-w, --working-dir` | Start shell in specified directory |
| `--write-config` | Write default config to a given path |
| `--enable-log-file` | Write logs to file in config directory |
| `--title-placeholder` | Start window with specified title |
| `--app-id` | Set Wayland app_id or X11 WM_CLASS (Linux/BSD) |
