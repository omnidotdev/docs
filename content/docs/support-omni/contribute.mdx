# Contribute Code

Omni welcomes contributions from the community. Here's how to get started.

## Getting Started

1. **Find a project**: Browse [github.com/omnidotdev](https://github.com/omnidotdev) for repositories
2. **Read the README**: Each project has setup instructions
3. **Check existing issues**: Look for `good first issue` or `help wanted` labels
4. **Join Discord**: Ask questions in [our Discord](https://discord.gg/omnidotdev)

## Contribution Guidelines

All Omni projects follow these guidelines:

### Code Style

- Use TypeScript where applicable
- Follow existing code patterns in the repository
- Write meaningful commit messages
- Add tests for new functionality

### Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run tests and linting
5. Submit a pull request with a clear description

### Code of Conduct

We expect all contributors to be respectful and constructive. See our [Code of Conduct](https://github.com/omnidotdev/.github/blob/master/CODE_OF_CONDUCT.md).

## Types of Contributions

- **Bug fixes**: Fix issues and improve stability
- **Features**: Add new functionality
- **Documentation**: Improve docs and examples
- **Tests**: Increase test coverage
- **Design**: UI/UX improvements

## Recognition

Contributors are recognized in:
- Repository CONTRIBUTORS files
- Release notes
- Our Discord community

## Questions?

- **GitHub Discussions**: Available on each repository
